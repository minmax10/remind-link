# Remind Link - 프로젝트 기획서

## 1. 프로젝트 비전

### 1.1 문제 정의
현대인들은 다양한 플랫폼에서 흥미로운 콘텐츠를 발견하고 저장하지만:
- 저장한 콘텐츠를 다시 찾기 어려움
- 여러 플랫폼에 분산되어 있어 통합 관리가 어려움
- 저장만 하고 다시 보지 않게 됨
- 카테고리별로 정리하는 것이 번거로움

### 1.2 솔루션
Remind Link는 사용자가 저장한 모든 콘텐츠를 자동으로 수집하고, AI를 활용해 지능적으로 분류하여 보기 좋게 정리해주는 서비스입니다.

## 2. 타겟 사용자

### 2.1 주요 타겟
- **콘텐츠 큐레이터**: 다양한 콘텐츠를 수집하고 정리하는 사람들
- **학습자**: 학습 자료를 체계적으로 관리하고 싶은 사람들
- **크리에이터**: 영감을 얻기 위해 다양한 콘텐츠를 수집하는 사람들
- **일반 사용자**: 북마크를 많이 저장하지만 다시 찾기 어려운 사람들

### 2.2 사용자 페르소나

#### 페르소나 1: 개발자 김철수 (28세)
- **행동**: 기술 블로그, GitHub 저장소, 개발 튜토리얼을 자주 저장
- **문제**: 저장한 자료를 다시 찾기 어려움, 카테고리별 정리가 안 됨
- **요구사항**: 기술 스택별, 난이도별로 자동 분류

#### 페르소나 2: 디자이너 이영희 (25세)
- **행동**: 인스타그램에서 디자인 영감을 많이 저장, Pinterest도 사용
- **문제**: 저장한 디자인을 다시 찾기 어려움, 여러 플랫폼에 분산
- **요구사항**: 스타일별, 색상별로 자동 분류, 시각적으로 보기 좋은 표시

#### 페르소나 3: 마케터 박민수 (32세)
- **행동**: 마케팅 케이스 스터디, 트렌드 자료를 자주 저장
- **문제**: 저장한 자료를 프로젝트별로 정리하고 싶음
- **요구사항**: 프로젝트별 태그, 날짜별 필터링

## 3. 핵심 기능 상세

### 3.1 콘텐츠 수집 기능

#### 3.1.1 클립보드 모니터링
**목적**: 사용자가 복사한 링크를 자동으로 감지하여 저장

**구현 방법**:
- 데스크톱 앱 또는 브라우저 확장 프로그램으로 클립보드 모니터링
- URL 패턴 감지 (http://, https://)
- 중복 링크 자동 필터링
- 사용자 확인 후 저장 (옵션: 자동 저장)

**기술적 고려사항**:
- OS별 클립보드 API 사용 (Windows: Win32 API, Mac: NSPasteboard, Linux: xclip)
- 실시간 모니터링 vs 주기적 체크
- 프라이버시: 클립보드 데이터는 로컬에서만 처리

#### 3.1.2 인스타그램 연동
**목적**: 인스타그램 저장글을 자동으로 동기화

**구현 방법**:
- Instagram Graph API 또는 웹 스크래핑
- OAuth 인증으로 사용자 계정 연동
- 주기적 동기화 (예: 1시간마다)
- 새로 저장한 글만 가져오기

**기술적 고려사항**:
- Instagram API 제한사항 이해
- 이미지 다운로드 및 저장
- 캡션 텍스트 추출

#### 3.1.3 쓰레드 연동
**목적**: 쓰레드(X) 저장글을 자동으로 동기화

**구현 방법**:
- Twitter/X API v2 사용
- OAuth 인증
- 북마크 API 활용
- 주기적 동기화

**기술적 고려사항**:
- Twitter API 비용 및 제한사항
- 이미지, 비디오 콘텐츠 처리

#### 3.1.4 브라우저 확장 프로그램
**목적**: 웹 브라우저에서 직접 북마크 추가 및 수집

**구현 방법**:
- Chrome Extension / Firefox Add-on
- 현재 페이지 정보 자동 추출 (제목, 설명, 이미지)
- 원클릭 저장 버튼
- 기존 브라우저 북마크 가져오기

### 3.2 AI 기반 자동 분류

#### 3.2.1 카테고리 분류
**카테고리 목록**:
1. 기술/개발 (Technology/Development)
2. 디자인/아트 (Design/Art)
3. 비즈니스/경제 (Business/Economics)
4. 뉴스/시사 (News/Current Events)
5. 엔터테인먼트 (Entertainment)
6. 교육/학습 (Education/Learning)
7. 건강/라이프스타일 (Health/Lifestyle)
8. 여행/음식 (Travel/Food)
9. 기타 (Others)

**구현 방법**:
- LLM (GPT-4, Claude)을 활용한 콘텐츠 분석
- 콘텐츠 제목, 설명, 메타데이터 분석
- 확률 기반 분류 (예: 기술 80%, 디자인 20%)
- 사용자 피드백으로 모델 개선

**프롬프트 예시**:
```
다음 콘텐츠를 분석하여 가장 적합한 카테고리를 선택해주세요.
제목: {title}
설명: {description}
URL: {url}

카테고리: 기술/개발, 디자인/아트, 비즈니스/경제, 뉴스/시사, 엔터테인먼트, 교육/학습, 건강/라이프스타일, 여행/음식, 기타

응답 형식: JSON
{
  "category": "카테고리명",
  "confidence": 0.95,
  "tags": ["태그1", "태그2", "태그3"],
  "summary": "콘텐츠 요약"
}
```

#### 3.2.2 태그 자동 생성
- 콘텐츠 내용 분석 후 관련 키워드 추출
- 사용자 정의 태그와 자동 태그 결합
- 태그 자동 완성 기능

#### 3.2.3 요약 생성
- 콘텐츠의 핵심 내용을 2-3문장으로 요약
- 읽기 시간 추정 (평균 읽기 속도 기준)

### 3.3 콘텐츠 표시 및 UI

#### 3.3.1 뷰 모드
1. **카드 뷰** (기본)
   - OG 이미지 또는 썸네일 표시
   - 제목, 설명, 태그 표시
   - 플랫폼 아이콘 표시
   - 저장 날짜 표시

2. **리스트 뷰**
   - 간결한 리스트 형태
   - 아이콘 + 제목 + 날짜
   - 빠른 스캔에 적합

3. **그리드 뷰**
   - Pinterest 스타일
   - 이미지 중심 표시
   - 반응형 그리드 레이아웃

4. **타임라인 뷰**
   - 저장한 날짜순으로 표시
   - 월별, 주별 그룹핑
   - "1주일 전", "1개월 전" 등 상대 시간 표시

#### 3.3.2 필터 및 정렬
- 카테고리별 필터
- 플랫폼별 필터 (인스타, 쓰레드, 클립보드 등)
- 태그별 필터
- 날짜 범위 필터
- 정렬: 최신순, 오래된순, 인기순(조회수)

#### 3.3.3 검색
- 전체 텍스트 검색 (제목, 설명, 태그)
- 자동 완성
- 검색 히스토리
- 고급 검색 (AND, OR 연산자)

### 3.4 콘텐츠 미리보기

#### 3.4.1 메타데이터 추출
- Open Graph 태그 파싱
- Twitter Card 메타데이터
- favicon 추출
- 대표 이미지 추출

#### 3.4.2 콘텐츠 요약
- AI 기반 요약 생성
- 핵심 키워드 하이라이트
- 읽기 시간 표시

## 4. 사용자 시나리오

### 시나리오 1: 개발자가 기술 블로그 저장
1. 개발자가 흥미로운 기술 블로그 링크를 발견
2. 링크를 복사 (클립보드에 저장)
3. Remind Link가 자동으로 링크 감지
4. 사용자에게 저장 확인 알림 표시
5. 사용자가 확인하면 자동으로:
   - 블로그 내용 분석
   - 카테고리: "기술/개발" 자동 분류
   - 태그: "React", "TypeScript", "튜토리얼" 자동 생성
   - 요약 생성
6. 나중에 "기술/개발" 카테고리에서 해당 글을 발견

### 시나리오 2: 디자이너가 인스타그램 저장글 확인
1. 디자이너가 인스타그램에서 디자인 영감을 저장
2. Remind Link가 1시간마다 인스타그램 저장글 동기화
3. 새로 저장한 디자인이 자동으로 추가됨
4. AI가 디자인 스타일 분석 후 카테고리 분류
5. 디자이너가 "디자인/아트" 카테고리에서 그리드 뷰로 확인
6. 색상, 스타일별로 필터링하여 원하는 디자인 찾기

### 시나리오 3: 마케터가 프로젝트별 자료 정리
1. 마케터가 여러 마케팅 케이스 스터디를 저장
2. 각 자료에 프로젝트별 태그 수동 추가
3. 프로젝트별로 필터링하여 관련 자료만 확인
4. 타임라인 뷰로 프로젝트 진행 과정 추적

## 7. 성공 지표 (KPI)

- **사용자 수**: 월간 활성 사용자 (MAU)

- **재방문율**: 주간/월간 재방문율
- **분류 정확도**: 사용자가 카테고리 수정하는 비율
- **사용자 만족도**: NPS (Net Promoter Score)
